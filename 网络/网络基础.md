  # 2. 认识协议
  
  ## 网络协议定义
  
  一句话：**网络协议是双方认可的结构体**：
  
  * 底层视角：
  
    网线/光纤里跑的是0/1比特流。两台机器间通信，要规定好比特流特定位代表什么
  
  * 代码视角：一段连续的内存布局规则
  
    ```cpp
    struct TCPHeader {
        uint16_t src_port;  // 源端口 (2字节)
        uint16_t dst_port;  // 目的端口 (2字节)
        uint32_t seq_num;   // 序列号 (4字节)
        uint32_t ack_num;   // 确认号 (4字节)
        // ... 其他字段
    };
    
    // 发送时：把 struct 强转成 char* 发出去
    send(sockfd, (char*)&my_header, sizeof(my_header), 0);
    
    // 接收时：把收到的 char* 强转回 struct 读取
    struct TCPHeader* header = (struct TCPHeader*)recv_buffer;
    ```
  
  ## 网络分层
  
  ![image-20260225105117775](https://gitee.com/binary-whispers/pic/raw/master///20260225105119890.png)
  
  OSI是权威机构定制的详细的网络分层结构，实际工程中常使用TCP/IP协议族分层结构，将上三层统一看作应用层
  
  > * **应用层：** 负责应⽤程序间沟通
  >   * **专注于用户需求，为用户提供应用功能**
  >   * **不关心数据如何传输，好比寄快递，我们只管把快递交给快递员，无需关心快递如何被运输**
  > * **传输层：**负责两台主机之间的数据传输
  >   * **TCP：**传输控制协议，安全可靠
  >   * **UDP：**只负责发送数据包，不保证数据包对方是否收到，如果想要安全可靠可以参考TCP的设计思想
  > * **网络层：**负责地址管理和路由选择
  > * **数据链路层：**负责设备之间的数据帧的传送和识别
  > * **物理层：**负责光电信号的传输方式
  
  
  
  **为什么要有TCP/IP协议族？**
  
  1. **为了解决异构系统的通信障碍**，世界上存在不同的硬件厂商，各自底层标准不同，TCP/IP制定了一份通用的语言，使得不同硬件可以通信
  
  2. **为了解决长距离通信的不可靠性**，与TCP/IP 协议族通过 IP 地址解决定位问题，通过 TCP 协议解决可靠性问题，本质上是一套应对复杂网络环境的解决方案
  
  3. **为了屏蔽底层物理差异**，它在应用层和物理硬件之间构建了一个抽象层，让开发者不需要关心底层是光纤还是 WiFi，实现了全球网络的逻辑统一
  
  # 3. 网络传输基本流程
  
  ## 局域网网络传输
  
  **MAC地址**
  
  > * MAC地址用来识别数据链路层中相连的节点，并且**标识主机的唯一性**
  > * 长度位`48`比特位，一般用`16`进制数字加冒号形式表示
  > * 通常是唯一的，出厂时就确定了，不能修改
  
  **通信方式**
  
  > 原理类似开会，会议室中只允许一个人讲话，一个人讲话所有人都能听到；若多人讲话，则会发生碰撞
  >
  > 现在A要给B发数据
  >
  > **正常通信：单车道畅通**
  >
  > 在早期的以太网中，所有人都连在同一根线上（集线器 Hub 模式），就像大家都在同一个会议室里 
  >
  > * **规则**：任何时刻，只允许一个人说话（发数据） 
  > * **流程**：主机 A 想发数据，先听一下线路上有没有人在发。如果安静，它就发
  > * **广播特性**：A 发出的声音，B、C、D 都能听见，但只有目标 B 会接收，其他人丢弃（根据 MAC 地址）
  >
  > ![image-20260225112355360](https://gitee.com/binary-whispers/pic/raw/master///20260225112357828.png)
  
  
  
  **发生碰撞：车祸现场**
  
  > 如果 A 和 C 同时也想发数据，并且都觉得刚才没人说话，于是同时开口，这就叫**数据碰撞（Collision）** 
  >
  > 这时候物理信号（电压）会在电缆上叠加，变成乱码，谁也听不懂
  
  
  
  ### 同一网段内两台主机发消息的过程
  
  这里我们省去物理层
  
  在同一个网段（比如都在家里的 WiFi 下），用户A发给用户B 一个“你好”，不需要经过路由器转发，直接通过**交换机**（或直连）进行 MAC 寻址。
  
  这个过程就是一次**自顶向下的“封装”和自底向上的“解包”**
  
  ![image-20260225114015094](https://gitee.com/binary-whispers/pic/raw/master///20260225114017079.png)
  
  
  
  **概念拆解：报头 VS 载荷 VS 报文：**
  
  利用快递信封这个例子理解：
  
  * 报文：整体，报文=报头+荷载，“信封+信纸”
  * 报头：信封，信封上写着寄件人、收件人、邮票等**控制信息**
  * 有效载荷：信纸，上层用户想真正传递的内容
  
  **上一层的报文，就是下一层的有效荷载**

  ## 跨网络传输
  
  ### IP地址
  
  `IP`有`IPv4`和`IPv6`两个版本，这里主要讨论`IPv4`：
  
  * `IP`地址是在`IP`协议中，**标识网络中不同主机的地址**
  * 对于`IPv4`，`IP`地址是一个`4`字节的整数
  * 通常用**点分十进制**字符串表示`IP`地址，例如`192.168.0.1`，用点分割的每一个数字表示一个字节，范围`[0,255]`
  
  **怎么出现两个地址了，怎么区分MAC地址和IP地址？**
  
  > * **IP地址->快递单上的终点收货地址**
  >   * 特点：**全程不变**，它决定了快递的**总体方向**
  > * **MAC地址->运输车辆的车牌号**
  >   * 特点：**每一程都在变**，它决定了快递在**一小段运输区间（局域网）** 交给谁
  >
  > 假设快递从北京到上海，途径济南：
  >
  > ![image-20260131175008831](https://i-blog.csdnimg.cn/img_convert/0f09a748a8e8b7f78963e88988ac36b3.png)
  
  **结论：**
  
  * IP地址（收货地址）：负责**宏观路径规划**
  * MAC（车牌号）：负责**微观局域网传输**

  ## 总结知识点
  
  * IP地址（网络层）
  
    * 定义：IP地址在网络中标识主机的唯一性
    * 作用：提供网络虚拟层，屏蔽底层物理网络差异
    * 特性：在路由转发过程中**IP地址保持不变**
  
  * MAC地址（数据链路层）
  
    * 定义：用来识别数据链路层中相连的节点，网卡出场时确定（物理性唯一）
    * 作用：局域网内定点传输
    * 特性：在跨网段传输中，**MAC地址逐跳改变**
  
  * 路由器（工作在网络层->实现从网络层到物理层的功能）
  
    * 功能：
      * 地址管理
      * 路由选择：规划两台主机之间的数据传输路线
      * 解包和封包：连接不同的数据链路，转换帧格式
  
  * 封装与分用：
  
    * 封装：自顶向下，每层加报头。应用层->TCP段->IP数据段->以太网帧
    * 分用：自底向上：每层去报头，根据报头中的协议字段决定交给上层的谁
