# 准备一张案例表

用三张表：

* `users`：用户表
* `conversations`：AI会话表
* `messages`：AI消息表

用户表：

```sql
create table users (
	id bigint primary key auto_increment,
    email varchar(100) not null unique,
    nickname varchar(50) not null,
    password_hash varchar(255) not null,
    avatar_url varchar(255) comment '头像，可以为空',
    created_at datetime not null default current_timestamp
);
```

AI会话表：

```sql
create table conversations (
	id bigint primary key auto_increment,
    user_id bigint not null comment '属于哪个用户',
    title varchar(100) not null default '新会话',
    created_at datetime not null default current_timestamp,
    updated_at datetime not null default current_timestamp
);
```

AI消息表：

```sql
create table messages (
	id bigint primary key auto_increment,
    conversation_id bigint not null,
    role varchar(20) not null comment '消息角色',
    content text not null comment '消息正文',
    created_at datetime not null default current_timestamp
);
```

# Create 新增数据

往表里新增一行业务数据

语法：

```sql
insert into 表名 (字段1,字段2,字段3...)
values (值1,值2,值3...)
```

注意：

* 字段和值要一一对应
* 字符串要用引号
* 自增主键不需要手写
* `not null`且没有默认值的字段必须写

# Read 查询数据

从表中按照条件找出需要的数据，查询时需要问三个问题：

* 从哪张表查？
* 查哪些字段？
* 按什么条件查？

## 查询所有字段

```sql
select * from 表名;
```

这种方式不推荐，因为：

1. 可能查出不需要的字段。
2. 可能查出敏感字段，如 `password_hash`。
3. 增加网络传输。
4. 表字段变化可能影响后端。
5. 如果没有 `WHERE` 和 `LIMIT`，才是全表返回，风险更大。

## 查询指定字段

```sql
select 字段1,字段2... from 表名;
```

## 根据ID查一条

## offset分页

跳过前面n条，再开始取数据

比如：

```sql
limit 5 offset 0
```

表示：从第1条开始，取5条

```sql
limit 5 offset 10
```

跳过前面10条，从第11条开始，取5条

那么分页计算即是：

第1页，每页五条：

```sql
mysql> select id, email, nickname from users order by created_at desc limit 5 offset 0;
+----+-----------------------+---------------+
| id | email                 | nickname      |
+----+-----------------------+---------------+
| 17 | ai_backend@163.com    | ai_backend    |
| 16 | mysql_learner@163.com | mysql_learner |
|  7 | aalalla@163.com       | aalalla       |
|  4 | lalla@163.com         | lalla         |
|  5 | lalla@163.cossm       | lalssla       |
+----+-----------------------+---------------+
5 rows in set (0.00 sec)
```

第2页，每页5条：

```sql
mysql> select id, email, nickname from users order by created_at desc limit 5 offset 5;
+----+-----------------+----------+
| id | email           | nickname |
+----+-----------------+----------+
|  7 | aalalla@163.com | aalalla  |
|  2 | kunkun@163.com  | kunkun   |
|  3 | luck@163.com    | luck     |
|  1 | vect@163.com    | vect     |
| 15 | go_dev@163.com  | go_dev   |
+----+-----------------+----------+
5 rows in set (0.00 sec)
```

出现重复的原因是只按 created_at 排序，而多条数据的 created_at 完全相同，导致相同时间内的记录顺序不稳定。分页查询必须使用稳定排序，所以应加上唯一字段 id 作为第二排序条件

```sql
mysql> select id, email, nickname from users order by created_at desc, id desc limit 5 offset 0;
+----+-----------------------+---------------+
| id | email                 | nickname      |
+----+-----------------------+---------------+
| 17 | ai_backend@163.com    | ai_backend    |
| 16 | mysql_learner@163.com | mysql_learner |
|  7 | aalalla@163.com       | aalalla       |
|  6 | lalla@163.coawdm      | lallzzzaaa    |
|  5 | lalla@163.cossm       | lalssla       |
+----+-----------------------+---------------+
5 rows in set (0.00 sec)

mysql> select id, email, nickname from users order by created_at desc, id desc limit 5 offset 5;
+----+----------------+----------+
| id | email          | nickname |
+----+----------------+----------+
|  4 | lalla@163.com  | lalla    |
|  3 | luck@163.com   | luck     |
|  2 | kunkun@163.com | kunkun   |
|  1 | vect@163.com   | vect     |
| 15 | go_dev@163.com | go_dev   |
+----+----------------+----------+
5 rows in set (0.00 sec)
```

分页公式：

```text
offset = (page - 1) * page_size
例如：
page = 3
page_size = 10
offset = (3 - 1) * 10 = 20
```

分页的核心是：**排序->限制条数->跳过规定数据**

# having的使用

having对分组统计后的结果进行二次过滤

区别：

```text
where：分组前过滤原始数据
having：分组后过滤统计结果
```

## 找出token超过3000的用户

先统计每个用户的token：

```sql
mysql> select user_id, sum(total_tokens) total_tokens from model_call_logs group by user_id;
+---------+--------------+
| user_id | total_tokens |
+---------+--------------+
|       1 |         4200 |
|       2 |         5500 |
|       3 |         2700 |
|       4 |         4000 |
|       5 |         3600 |
+---------+--------------+
5 rows in set (0.00 sec)

```

然后只保留总数大于3000的用户：

```sql
mysql> select user_id, sum(total_tokens) total_tokens from model_call_logs group by user_id having sum(total_tokens) > 3000;
+---------+--------------+
| user_id | total_tokens |
+---------+--------------+
|       1 |         4200 |
|       2 |         5500 |
|       4 |         4000 |
|       5 |         3600 |
+---------+--------------+
4 rows in set (0.02 sec)

```

# SQL执行顺序

SQL书写的顺序如下：

```text
SELECT
FROM
WHERE
GROUP BY
HAVING
ORDER BY
LIMIT
```

逻辑执行的顺序如下：

```sql
FROM		# 从哪张表
WHERE		# 筛选原始行数据
GROUP BY	# 分组原始行数据
HAVING		# 过滤查询数据
SELECT		# 针对目标字段
ORDER BY	# 最后排序
LIMIT		# 已经有结果了，限制结果返回数量
```
