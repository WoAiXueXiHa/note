# 什么是指针？

指针：**本质还是变量，这个变量存的是内存地址**--->对比普通变量，普通变量存的是值

**指针是为了能直接或间接操作内存**

指针大小：**64位8字节，32位4字节**--->地址总线决定 $2^{64}$和$2^{32}$寻址能力--->和指针类型无关，是系统定死的

****



# 指针的基本操作

`&`取地址

```cpp
int a = 10;
int* ptr = &a;
```

`&a`取变量`a`的地址

`*`解引用： **根据`ptr`里存的地址，找到那块内存中的值**

* `ptr`是地址
* `*ptr`是地址对应的内容

而`ptr = &a`，那么`*ptr`就是`a`

所以，`*ptr = 30` 等价于 `a = 30`

****

> 指针类型决定指针步长，页决定了解引用时如何解释那块内存

```cpp
int a = 10;
int* ptr = &a;
```

`int*` **告诉编译器：这个地址存的是一个`int`类型对象**

那么编译器就知道了：

* 解引用后应该按照什么类型解释那块内存
* 指针加减一步跨越几个字节

例如：

* `char*` 加 1，前进 1 字节，解引用按1字节解释

* `int*` 加 1，前进 4 字节，解引用按4字节解释

****



# 空指针、野指针、悬空指针

> 空指针：**不指向任何有效对象**的指针

特点：空指针合法，但是不能解引用

> 为什么不能解引用空指针？--->**解引用就是一次内存访问**

常规解引用普通指针会发生：

* CPU执行`load`指令，要去读这个地址的内容
* CPU的MMU根据页表把**虚拟地址映射到真实物理地址**
* 同时检查：
  * 这个页是否存在
  * 是否有权限读
* 检查失败，触发异常，OS接管异常，向进程发送信号，杀死进程，程序表现为**段错误**

而解引用空指针：**CPU要访问地址0附近的内存**

而操作系统**不会映射`0x0`附近的低地址页，解引用空指针，找不到映射**，MMU查页表找不到，触发缺页异常，内核处理异常杀死进程

****

> 野指针：**没有明确指向合法对象的非法指针**

典型的是未初始化：

```cpp
int* p;
*p = 10;
```

`p`里存的是随机地址，这个地址可能是**未映射区域**或者**某块已经映射的内存**--->未定义行为，操作野指针不知道之后会发生什么，可能崩溃，可能结果不是预期，可能正常运行

> 为什么有的野指针不崩溃，反而更危险？

```cpp
int a = 10;
int* p;
*p = 20;
```

`p`这个垃圾地址可能刚好指向：

* 当前进程栈上的某个位置
* 堆里某个已经分配的区域
* 某个全局变量

CPU访问没有触发异常，程序继续跑，但是：

* 修改了不该修改的内容
* 破坏了栈帧
* 埋下更晚爆炸的雷

所以：**段错误反而是幸运的，非法访问系统被即使拦截了，最可怕的是访问了错误但可访问的地址，并且操作了它**

所以，定义时要初始化

****

> 悬空指针：**指针是合法地址，但是原来指向的对象生命周期已经结束，地址本身可能暂时还可访问，所以不一定立刻崩溃，但对象语义已经失效了**

```cpp
int* p = new int(10);
delete p;
*p = 20;	// 使用了悬空指针

int* func() {
    int a = 10;
    return &a;	// 返回时，a的生命周期结束
}
```

所以，`delete`指针之后要立马置空

****



# 指针和引用

引用的本质是**变量的别名**

```cpp
int a = 10;
int& ra = a;
```

`ra`就是`a`的另一个名字，一个人，大名和小名

**核心区别：**

* 是否必须初始化？

  * 指针可以不初始化--->但是建议初始化
  * 引用必须初始化

* 能否改变指向？

  * 指针可以改变指向

  * 引用一旦绑定不可修改

    ```cpp
     int a = 10;
        cout << "a = " << a << endl;
        int b = 20;
        int& ra = a;
        ra = b;         // 这里是给ra赋值，而不是让ra指向b
        cout << "a = " << a << endl;
        cout << "b = " << b << endl;
        cout << "ra = " << ra << endl;
        return 0;
    ```

* 是否可以为空？

  * 指针可以
  * 引用不行

**引用底层借助指针实现，语言语义上有区别，指针是独立对象，引用是别名**

****



# 指针和数组

> 数组名是什么？

`int arr[5] = { 1, 2, 3, 4, 5};`

大多数情况，`arr`会退化成为指向首元素的指针

`int* p = arr;`合法

注意两个特例：

* `sizeof(数组名)`是整个数组的大小
* `&数组名`取出的是整个数组地址

还需要注意一个语法糖：

`arr[2]`等价于`*(arr + 2)`

对于二维数组`arr[1][2]`等价于`*(*(arr + 1) + 2)`

> 指针数组和数组指针

指针数组：**装指针的数组，本质是数组，一段连续的内存**

`int* arr[10]`

数组指针：**指向数组的指针，本质是指针，存数组的地址**

`int (*p)[5]`

# 指针函数和函数指针

函数指针：**指向函数的指针**

```cpp
int add(int a, int b){ return a + b; }
int (*fp)(int, int) = add;
```

**`返回类型 (*指针变量名)(参数类型) = 函数名`**

指针函数：**返回值是指针的函数**

`int* func() { static int a = 10; return &a; }`
