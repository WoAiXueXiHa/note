# 对象模型

## 类的概述和大小

类的本质是两部分：

* 数据：成员变量，最后都要放进对象内存中
* 行为：成员函数，**普通成员函数不放进对象中**，代码区只有一份，对象调用它时会传`this`指针

**对象里放的是成员变量，成员函数代码通常不在对象里**

```cpp
class A {
public:
    int x;
    char c;
    
    void f() {}
};
```

定义：`A a;`

对象内存中有：

```txt
x
c
对齐填充字节
```

**没有`f`函数代码**，调用时会类似`A::f(&a);`，把对象地址当作`this`传进去

**普通成员函数不占对象大小**

> ==空类大小是多少？==

空类大小为1字节，**要保证不同对象有不地址**，这一个字节是标识我这个类存在，占位

## this指针

每个非静态成员函数，都会有一个隐式的`this`指针

`void A::f(A* const this)`

所以：

* 非静态成员函数能直接访问成员变量，通过`this`指针
* 静态成员函数不能直接访问非静态成员变量，没有`this`指针

## 静态成员放哪里？

```cpp
class A {
public:
    int x;
    static int y;
};
```

`A a;`对象`a`里有`x`，没有`y`

**静态成员属于类，不独属于某个对象**，静态成员在**全局/静态区**单独存放一份

**对象内存里只放非静态成员变量，不放静态成员变量**

# 继承模型

## 继承后的内存长什么样？

```cpp
class A {
public:
    int a;
};
class B : pubilc A {
public:
    int x;
}
```

定义：`B b;`

`b`对象的内存布局为：

```txt
[父类A的对象a]
[子类B的成员x]
```

**子类对象内存里，先放父类的成员对象，再放自己新增的成员**--->**先有父，后有子**

> ==为什么父类指针能调用子类==

例如：

```cpp
B b; // 子类对象
A* pa = &b;	// 父类指针指向子类对象
```

子类对象开头有完整的父类成员分布，把子类的起始地址当成父类的地址是合理的

> ==继承后成员访问怎么实现==

子类对象`b`的内存分布大致是：

```txt
[a]
[x]
```

* `b.a`去前面部分
* `b.x`去后面部分

**非虚成员访问，是对象首地址+偏移量**
