# C++程序的内存布局

![image-20260404155428443](https://gitee.com/binary-whispers/pic/raw/master///20260404155430821.png)

* 代码段：主要存放**机器指令和函数代码**--->CPU取指令执行的区域，只读，可执行
* `.rodata`只读数据段：主要存放**字符串常量、只读常量**--->存放程序运行期间不应该被修改的数据，数据会被频繁读，做只读保护
* `.data`已初始化数据段：主要存放**已初始化的全局变量/静态变量**--->整个程序运行期间都存在，有明确的初始值，可读可写
* `.bss`未初始化数据段：存放**未初始化或初始化为0的全局/静态变量**--->只记录这里需要多大一块0初始化的空间，占位
* 堆区：存放**手动申请的对象**--->**提供运行时动态申请的大块可管理内存**，向上生长
* 栈区：存放**局部对象，函数参数，栈帧信息**--->**支撑函数调用**，每调用一个函数，就压入一个新的栈帧，函数返回，栈帧弹出

# new/delete 和 malloc/free

> 本质区别

`malloc/free`：

* C风格，函数
* 只**分配/释放原始内存**
* **不调用构造/析构**

`new/delete`：

* C++风格，运算符
* `new`：**分配内存+构造对象**
* `delete`：**析构对象+释放内存**

> `malloc`的行为：**给用户一块`size`字节的可用空间**

`int* p = (int*)malloc(sizeof(int));`

1. 向堆内存申请一块空间
2. 返回这块空间的首地址
3. 这块空间是未初始化的垃圾值

> `free`的行为

`free(p)`

1. 把这块内存释放
2. 标记后续可以复用

**不会调用析构**

> `new`的行为

`A* p = new A();`

1. **调用`operator new`，`operator new`又会调用`malloc`分配原始内存**
2. **在内存上构造对象**

> `free`的行为

`delete p;`

1. **调用析构函数**
2. **调用`operator delete`释放内存**

# new[]/delete[]和 malloc/free的区别

> `new[]`的行为

`A* arr = new A[10];`

1. 申请内存，**会多申请一块隐藏头部空间（4or8字节），记录数组有多少个元素**
2. 创建10个对象，调用10次构造

> `delete[]`的行为

`delete[] arr;`

1. 读取头部空间，知道有10个对象
2. **倒序调用10次析构**
3. 释放申请的空间

# static关键字

`static`核心作用是**改变对象存储期/链接属性**

> 修饰局部变量

```cpp
void func() {
    static int cnt = 0;
    cnt++;
}
```

* 作用域仍在函数内
* 生命周期变为整个程序的生命周期
* 只能初始化一次

> 修饰全局变量

`static int g = 10;`

* **链接属性变为内部链接->只在当前源文件内可见**


> 修饰类成员变量

```cpp
class A {
public:
    static int cnt;
};
```

* 属于类本身，不属于某个对象
* 所有类对象共享一份

> 修饰类成员函数

```cpp
class A {
public:
    static void func();
};
```

* 不依赖具体对象
* 没有`this`指针
* 只能访问静态成员变量

总结：**对局部变量，延长生命周期**  **对全局变量/函数，限制外部可见**   **对类成员，表明属于类本身，而不是某个特定对象**

# 内存对齐

> 什么是内存对齐

**对象起始地址要满足某种整数倍的要求**

例如一个`int`要求4字节对齐，那么最后的地址就是4的倍数

> 为什么要内存对齐？

**保证CPU访问效率**，CPU和内存是通过**数据总线通信**，64位OS每次必须读取**8或者8的整数倍**

就比如一个`double`在地址`3~10`之间

CPU为了读`double`：

* 先读`0~7`
* 再读`8~15`
* 把两次读到的数据各种处理，效率太低

> 内存对齐规则

前提：编译器都有默认的**对齐数**，64位默认为8

规则一：**成员自身对齐（决定字段偏移量）**

$min(自身类型大小，默认对齐数)$的整数倍

规则二：**结构体整体对齐（决定结构体最终大小）**

结构体大小=$min(内部最大基础成员的大小，默认对齐数)$的整数倍

例如：64位系统

```cpp
struct A {
    char c;
    int b;
    double c;
}
```

```txt
c _ _ _ b b b b c c c c c c c c
0 1 2 3 4 5 6 7 8 .......    15
```

最终大小为16字节
