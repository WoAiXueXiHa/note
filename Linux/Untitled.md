# 并发编程解决什么问题？

**1. 资源竞争：多个线程同时改一份数据**	

* 多个线程同时`cnt++`->抢票冲突
* 多个线程同时往队列中`push/pop`
* 多个线程同时改连接状态

导致：

* 数组错乱
* 覆盖更新
* 数据不一致

**锁**解决

**2. 协作/等待：线程需要等一个条件成立**

* 消费者线程要等队列非空
* 生产者线程要等队列非满
* 一个线程要等任务到来

这是需要：

* 条件不满足睡眠
* 条件满足被唤醒

**条件变量**解决

总结：

> **锁解决同时动，条件变量解决没到时候别动**

# 为什么cnt++不安全？

## cnt++不是原子操作

**原子操作：要么不执行，要么一口气执行完，不会被打断**

`cnt++`：

* 读`cnt`
* `cnt += 1`
* 更新`cnt`数值

三个步骤

两个线程同时执行：

* A读到5，B也读到5
* A写6，B也写6

以为加了两次，其实最终只加了一次

## 什么是临界区

**临界区是访问共享资源的那段代码**

`cnt++;`

```cpp
if(!queue.empty()) {
    auto x = queue.front();
    queue.pop();
}
```

只要多个线程都能进来改一份相同的资源，这就是临界区
