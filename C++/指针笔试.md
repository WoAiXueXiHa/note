### 笔试题 1：基础步长欺骗

**代码核心：** `int a[5] = {1,2,3,4,5};` `int *ptr = (int *)(&a + 1);` `printf("%d,%d", *(a + 1), *(ptr - 1));`

**架构师拆解（底层逻辑）：**

1. `*(a + 1)`：数组名 `a` 退化为指向首元素 `1` 的指针。`a + 1` 步长是一个 `int`（4字节），指向 `2`。解引用拿到 **2**。
2. `&a + 1`：`&a` 是**数组指针**，步长是整个数组（20字节）。`&a + 1` 直接跳到了数组末尾的下一个内存位置。
3. `ptr - 1`：`&a + 1` 被强转成了 `int*` 并赋给 `ptr`。此时 `ptr` 的步长变回了 4 字节。`ptr - 1` 往回退 4 个字节，刚好指在数组最后一个元素 `5` 上。解引用拿到 **5**。

* **运行结果**：`2, 5`
* **考察点**：`a` 与 `&a` 步长的本质区别。

### 笔试题 2：指针与整数加法的降维打击

**代码核心：** `p` 是结构体指针，大小20字节，初值 `0x100000`。 打印 `p + 0x1`，`(unsigned long)p + 0x1`，`(unsigned int*)p + 0x1`。

**架构师拆解（底层逻辑）：**

1. `p + 0x1`：这是**指针加法**。步长是 `sizeof(struct Test)` 即 20 字节（十六进制为 `0x14`）。所以 `0x100000 + 0x14` = **0x100014**。
2. `(unsigned long)p + 0x1`：指针被强转成了**普通整数**！这就成了纯粹的数学加法。`0x100000 + 1` = **0x100001**。
3. `(unsigned int*)p + 0x1`：指针被强转成了 `unsigned int*`。步长变成了 `sizeof(unsigned int)` 即 4 字节。所以 `0x100000 + 0x4` = **0x100004**。

* **运行结果**：`0x100014, 0x100001, 0x100004` (实际打印可能会根据编译器补齐前导0)
* **考察点**：强转会改变指针的“导航仪”步长，转成整数则彻底失去指针特性。

### 笔试题 3：最硬核的内存切片与大小端

**代码核心：** `int a[4] = {1, 2, 3, 4};` `int *ptr1 = (int *)(&a + 1);` `int *ptr2 = (int *)((int)a + 1);`

**架构师拆解（底层逻辑）：**

1. `ptr1[-1]`：等价于 `*(ptr1 - 1)`。跟第一题一样，`ptr1` 指向数组末尾，减 1 回退一个 `int`，指向 `4`。以 `%x` 打印是 **4**。
2. `*ptr2`（**致命陷阱**）：
   * 假设系统是小端字节序（低位数据在低地址，x86标配）。数组前两个元素的内存字节排布如下： `a[0]: 01 00 00 00` `a[1]: 02 00 00 00`
   * `(int)a + 1`：把起始地址转成整数并加 1，这意味着地址**在物理内存上只前进了 1 个字节**！
   * `(int*)`：强转回 `int*`，此时指针指向了 `a[0]` 的第二个字节。
   * 解引用 `*ptr2`：从这里往后连读 4 个字节，读出来的是 `00 00 00 02`。
   * 小端机器解析这四个字节，拼出来的十六进制数就是 **0x02000000**。

* **运行结果**：`4, 2000000`
* **考察点**：字节序（Endianness）与字节级内存偏移。

### 笔试题 4：逗号表达式的障眼法

**代码核心：** `int a[3][2] = { (0, 1), (2, 3), (4, 5) };` `p = a[0]; printf("%d", p[0]);`

**架构师拆解（底层逻辑）：**

* **极其致命的语法陷阱**：大括号里面套的不是花括号 `{}` 而是圆括号 `()`！这触发了 C 语言的**逗号表达式**。
* 逗号表达式 `(A, B)` 的结果永远是最右边的 `B`。
* 所以 `(0, 1)` 结果是 `1`，`(2, 3)` 结果是 `3`，`(4, 5)` 结果是 `5`。
* 数组真正初始化的样子是：`a[3][2] = {1, 3, 5, 0, 0, 0};`
* `p = a[0]` 指向第一行 `{1, 3}`。`p[0]` 就是第一个元素 **1**。
* **运行结果**：`1`
* **考察点**：细心程度与逗号表达式陷阱。

### 笔试题 5：高维指针降维计算

**代码核心：** `int a[5][5]; int(*p)[4] = a;` 打印 `&p[4][2] - &a[4][2]`

**架构师拆解（底层逻辑）：**

* 我们在内存中把二维数组拉平：
* `a` 认为每行有 5 个元素。`&a[4][2]` 的物理偏移量是 `4 * 5 + 2 = 22`（即第 22 个元素）。
* `p` 认为每行只有 4 个元素。`&p[4][2]` 的物理偏移量是 `4 * 4 + 2 = 18`（即第 18 个元素）。
* 指针相减，得到的是**相差的元素个数**：`18 - 22 = -4`。
* 以 `%p`（十六进制）打印 `-4` 的内存补码就是 **FFFFFFFC**（假设32位）。以 `%d` 打印就是 **-4**。
* **运行结果**：`FFFFFFFC, -4`
* **考察点**：二维数组底层是一维连续内存，指针类型决定了“换行”的跨度。

### 笔试题 6：解引用降维

**代码核心：** `int aa[2][5] = {1,2,3...10};` `int *ptr1 = (int *)(&aa + 1);` `int *ptr2 = (int *)(*(aa + 1));`

**架构师拆解（底层逻辑）：**

1. `ptr1`：`&aa` 是整个矩阵的地址，步长 40 字节。`+1` 跳过整个矩阵。`-1` 退回一个元素，拿到最后的 **10**。
2. `ptr2`：`aa` 是矩阵首行地址，`aa + 1` 是第二行的地址（步长一行）。重点来了：**`\*(aa + 1)` 将“行指针”解引用，降维成了“指向第二行首元素的普通 `int\*` 指针”**（指向 6）。`ptr2 - 1` 往回退一个元素，拿到第一行的最后一个 **5**。

* **运行结果**：`10, 5`
* **考察点**：二维数组名 `+1` 与 `&数组名 + 1` 的区别，以及 `*` 的降维作用。

### 笔试题 7：指针数组的基础寻址

**代码核心：** `char *a[] = {"work", "at", "alibaba"};` `char** pa = a; pa++; printf("%s", *pa);`

**架构师拆解（底层逻辑）：**

* `a` 是一个货架，上面放了三个字符串的首地址。
* `pa` 是一个二级指针，初始指向 `a[0]`（存着 "work" 的地址）。
* `pa++`，`pa` 往下走一格，指向了 `a[1]`。
* `*pa` 拿到了 `a[1]` 里的内容（即 "at" 的首地址）。打印字符串得到 **at**。
* **运行结果**：`at`
* **考察点**：二级指针的基本步长偏移。

### 笔试题 8：阿里终极 Boss 题（状态机推演）

**代码核心：** `char *c[] = {"ENTER", "NEW", "POINT", "FIRST"};` `char **cp[] = {c+3, c+2, c+1, c};` `char ***cpp = cp;`

**架构师拆解（请在脑海里画出三个货架的连线）：** 初始状态：`cpp` 指向 `cp[0]`。

1. `**++cpp`：`cpp` 先自增，指向 `cp[1]`。解引用拿到 `c+2`，再解引用拿到 `"POINT"`。 *(此时 `cpp` 停在 `cp[1]`)*
2. `*--*++cpp+3`（**极其变态，按优先级拆**）：
   * `++cpp`：`cpp` 前进到 `cp[2]`。
   * `*++cpp`：拿到 `cp[2]` 的内容（`c+1`）。
   * `--`：**副作用发作**！将 `cp[2]` 里的内容 `c+1` 自减变成了 `c`。
   * `*`：解引用 `c`，拿到 `"ENTER"` 的首地址。
   * `+3`：字符串指针前进 3 个字母，停在 'E' 上，打印出 **ER**。 *(此时 `cpp` 停在 `cp[2]`)*
3. `*cpp[-2]+3`：
   * `cpp[-2]` 等价于 `*(cpp - 2)`。`cpp` 当前在 `cp[2]`，减 2 就是拿 `cp[0]` 的内容（`c+3`）。
   * `*(c+3)` 拿到 `"FIRST"`。
   * `+3` 前进 3 个字母，打印出 **ST**。 *(此时 `cpp` 依然停在 `cp[2]`)*
4. `cpp[-1][-1]+1`：
   * `cpp[-1]` 拿到 `cp[1]` 的内容（依然是 `c+2`）。
   * `(c+2)[-1]` 等价于 `*(c+2 - 1)`，即 `*(c+1)`。
   * **注意**：由于第 2 步的副作用，很多人会搞错，但那个副作用修改的是 `cp[2]`，`cp[1]` 还是安全的 `c+2`！所以这里拿到的是 `"NEW"`。
   * `+1` 前进 1 个字母，打印出 **EW**。

* **运行结果**：`POINT`, `ER`, `ST`, `EW`
* **考察点**：多级指针的拓扑结构、操作符优先级、以及 `++`/`--` 在复杂表达式中的永久副作用。
