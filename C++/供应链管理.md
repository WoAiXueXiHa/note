## 一、供应链基础理论（理论基石，易考选择 / 简答）

### 1. 供应链核心定义（双权威版本，必背）

| 定义来源                           | 核心内容                                                     |
| ---------------------------------- | ------------------------------------------------------------ |
| 《物流术语》国标（GB/T18354-2006） | 生产及流通过程中，为将产品 / 服务交付最终用户，由上下游企业共同建立的**需求链状网** |
| 美国供应链管理学会                 | 支持产品 / 服务从供应者到最终使用者的**全生命周期过程**，涵盖物流、资金流、信息流、知识流 |

### 2. 供应链的 4 大特征（易考简答题，需结合例子）

| 特征     | 核心解释                                                     | 例子                                                         |
| -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 复杂性   | 节点企业多（如苹果供应链含上千供应商），结构比单个企业复杂，需用复杂系统理论解决 | 汽车供应链涉及原材料供应商、零部件厂商、组装厂、经销商等十余个环节 |
| 动态性   | 节点企业随战略 / 市场需求动态更新，不可静态分析              | 某手机厂商因某供应商产能不足，替换为新供应商                 |
| 面向需求 | 供应链的形成、重构均基于市场需求，最终服务单个用户，需求有随机性 | 羽绒服供应链冬季扩张、夏季收缩，按需调整库存                 |
| 交叉性   | 节点企业可属于多个供应链，形成交叉结构，增加协调难度         | 某包装厂既为食品供应链供货，也为日化供应链服务               |

### 3. 供应链一体化：纵向 vs 横向（对比类必考）

| 类型       | 定义                                                         | 优缺点                                                       | 适用场景                             |
| ---------- | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------ |
| 纵向一体化 | 企业覆盖供应链多环节（如制造商自产原材料、自办零售），“大而全” | 优点：控制上下游，减少依赖；缺点：成本高、灵活性差、资源分散 | 传统稳定市场（如 20 世纪初的钢铁厂） |
| 横向一体化 | 聚焦核心业务，非核心外包（如苹果外包生产给富士康），形成 “节点协同链” | 优点：专注核心竞争力、成本低、响应快；缺点：需强协调能力     | 现代多变市场（如电子、快消）         |

### 4. 波特 “价值链” 理论（供应链管理的理论基础，易考启示）

#### （1）价值链构成（9 种活动，分两类）

| 活动类型 | 核心作用                   | 具体内容（制造业案例）                                       |
| -------- | -------------------------- | ------------------------------------------------------------ |
| 基本活动 | 直接创造价值               | 进货后勤（原材料运输）、生产作业（零件加工）、出货后勤（成品配送）、市场营销（广告推广）、服务（售后维修） |
| 辅助活动 | 支持基本活动，间接创造价值 | 采购（原材料采购）、技术开发（生产工艺升级）、人力资源管理（员工培训）、企业基础设施（财务 / 法务） |

#### （2）对供应链管理的 3 大启示（必背）

1. **竞争优势藏于环节**：企业竞争优势不是整体优势，而是价值链单个 / 多个环节的优势（如苹果的优势是 “设计 + 供应链协调”，而非生产环节）；
2. **环节集成是关键**：需协调自身价值链各环节（如采购与生产协同），或与其他企业联盟（如苹果与三星合作供应芯片），才能形成竞争优势；
3. **价值链嵌套于更大系统**：企业价值链需与供应商、客户的价值链融合（如苹果价值链 + 富士康生产价值链 + 京东销售价值链 = 苹果完整供应链）。

## 二、供应链管理核心内涵（理解类考点，易考多选）

### 1. 5 大基本内涵（需理解每个内涵的意义）

| 内涵                       | 核心解释                                                     | 关键作用                                  |
| -------------------------- | ------------------------------------------------------------ | ----------------------------------------- |
| 以顾客满意度为目标         | 所有活动围绕 “满足顾客需求” 展开（如京东 “211 限时达”）      | 供应链的终极目标，决定服务方向            |
| 物流 / 信息流 / 资金流集成 | 三大流需跨企业同步（如沃尔玛与宝洁共享销售数据，同步补货）   | 避免 “信息孤岛”，减少需求扭曲（牛鞭效应） |
| 重视物流企业参与           | 物流周期比制造周期更关键（如服装制造 1 天，物流配送可能需 3 天） | 缩短整体供应链周期，提升响应速度          |
| 资源外用（外包）           | 非核心业务外包（如耐克外包生产，聚焦设计 / 营销）            | 降低成本，专注核心竞争力                  |
| 借助信息技术               | 依赖 EDI/Internet/ERP 等技术（如亚马逊用 AI 预测需求）       | 实现信息实时共享，支撑流程协同            |

## 三、供应链绩效评价（重点模块，易考阐述 / 计算）

### 1. 平衡记分卡（4 维度 + 4 大平衡，必考阐述题）

#### （1）4 维度指标设计（需结合企业案例，如麦当劳）

| 维度         | 核心目标               | 供应链关键指标（举例 + 解释）                                |
| ------------ | ---------------------- | ------------------------------------------------------------ |
| 财务维度     | 控制成本、提升盈利     | ①供应链总成本：运输 + 库存 + 设施成本（如麦当劳冷链运输成本占比）；②库存周转率：年销售成本 / 平均库存（反映库存效率）；③总资产报酬率：利润 / 平均总资产（衡量整体盈利） |
| 客户维度     | 满足需求、提升满意度   | ①准时交货率：按时送达订单占比（麦当劳门店原料 24 小时内送达比例）；②完美订单率：无缺货 / 无破损的订单占比；③客户抱怨率：投诉订单数 / 总订单数 |
| 内部流程维度 | 优化核心流程效率       | ①订单总周期：从客户下单到收货的时间（如麦当劳汉堡原料从供应商到门店的周期）；②产销率：实际产量 / 需求产量（反映供需匹配）；③物流货损率：破损货物量 / 总运输量 |
| 学习成长维度 | 持续改进、提升创新能力 | ①信息共享程度：与供应商共享需求数据的频率（如麦当劳与农场共享食材需求）；②新产品销售占比：新品收入 / 总营收；③员工培训费用：年均人均培训成本（提升员工能力） |

#### （2）4 大平衡关系（易考简答题）

1. **过去 - 现在 - 未来平衡**：财务（过去成果）、客户 / 流程（现在运营）、学习成长（未来潜力）；
2. **短期 - 长期平衡**：财务（短期盈利）、学习成长（长期竞争力）；
3. **内部 - 外部平衡**：内部流程（内部效率）、客户（外部需求）；
4. **结果 - 动因平衡**：财务（结果指标）、客户 / 流程 / 学习成长（动因指标，驱动结果实现）。

| -------------- | ------------------------------ | ------------------------------------------------------------ |
| 面向顾客的指标 | 顾客视角（供应链对顾客的价值） | 可靠性（准时交货率）、反应速度（订单周期）、柔性（按需调整产量的能力） |

## 四、供应链运营战略（战略层考点，易考对比 / 匹配）

### 1. 供应链运营战略 3 大目标

| 目标     | 核心解释                                             | 衡量指标                           |
| -------- | ---------------------------------------------------- | ---------------------------------- |
| 成本最小 | 降低供应链总变动成本（运输 + 库存成本）              | 单位产品运输成本、库存持有成本占比 |
| 资本最小 | 减少对物流系统的直接投资（如少建仓库，用第三方物流） | 物流设施投资金额、固定资产占比     |
| 服务改善 | 提升顾客需求满足率                                   | 订单满足率、缺货率                 |

### 2. 2 种基本运营战略（对比类必考，表格记忆）

| 战略类型         | 首要目标       | 产品设计战略           | 价格战略               | 制造战略             | 库存战略         | 运输战略                      |
| ---------------- | -------------- | ---------------------- | ---------------------- | -------------------- | ---------------- | ----------------------------- |
| 高效率供应链     | 低成本满足需求 | 最小成本，最大化绩效   | 价格为核心，边际利润低 | 高效利用设备降成本   | 最小化库存降成本 | 依赖低成本方式（公路 / 铁路） |
| 高响应能力供应链 | 快速响应需求   | 模块化设计（延迟差异） | 价格非核心，边际利润高 | 保持柔性应对意外需求 | 保留缓冲库存     | 依赖快速方式（空运 / 快递）   |

### 3. 战略匹配方法（3 步走，易考简答题）

1. **理解顾客**：明确目标顾客的需求差异（如订购数量、响应时间、产品品种），判断**潜在需求不确定性**（需求越不确定，需越高响应能力）；
2. **理解供应链**：供应链有 2 大功能 ——①物理功能（低成本运输 / 生产）；②市场中介功能（快速响应需求），需在 “反应能力” 与 “赢利水平” 间权衡；
3. **获取匹配**：使供应链响应能力与顾客需求不确定性匹配（如快时尚品牌 ZARA 需 “高响应能力供应链”，而矿泉水企业需 “高效率供应链”）。

## 五、供应链网络设计（运营层考点，易考计算 / 案例）

### 1. 网络选址 5 大决策内容（易考多选 / 简答）

| 决策内容        | 核心问题                                                     | 关键影响                                                     |
| --------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 设施区位        | 选 “区域”（如华东 / 华南）+ 选 “具体位置”（如靠近港口 / 高速） | 影响运输成本（靠近港口降低海运成本）、劳动力成本（中西部劳动力廉价） |
| 设施数目        | 建多少个仓库 / 工厂                                          | 数目多：响应快但成本高；数目少：成本低但响应慢（如京东在全国建 30 + 区域仓） |
| 设施功能        | 每个设施的作用（如仓库是 “存储” 还是 “转运”）                | 存储型仓库需大空间，转运型仓库需靠近运输枢纽                 |
| 市场 / 供给配置 | 每个设施服务哪些客户？从哪些供应商进货                       | 避免设施服务重叠，降低运输距离（如上海仓服务长三角客户）     |
| 设施容量        | 设施的最大处理能力（如工厂年产 100 万件）                    | 容量过高：设备闲置；容量过低：需求无法满足（如某口罩厂疫情期间扩容） |

### 2. 多级供应链系统设计（工厂 - 分销中心 - 市场，必考计算题）

#### （1）经典案例条件

* 2 个工厂（P1 无产能限制，P2 产能 60000 件）；2 个分销中心（W1、W2）；3 个市场（C1 需求 50000 件、C2 需求 100000 件、C3 需求 50000 件）；
* 单位运输成本（P→W→C）：P1→W1=0，P1→W2=5；P2→W1=4，P2→W2=2；W1→C1=3，W1→C2=4，W1→C3=5；W2→C1=2，W2→C2=1，W2→C3=2。

#### （2）3 种求解方法（重点掌握线性规划法）

| 方法                       | 核心逻辑                                                     | 计算结果（总成本）    |
| -------------------------- | ------------------------------------------------------------ | --------------------- |
| 方法一（逐环节选最低成本） | C1/C2/C3 均由 W2 供货，P2 供 W2 60000 件，P1 供 W2 140000 件 | 1120000 元            |
| 方法二（分市场选最优路径） | C1 由 W1 供货，C2/C3 由 W2 供货，P2 供 W2 60000 件，P1 供 W2 90000 件 | 920000 元             |
| 方法三（线性规划优化）     | 目标函数：\(\text{Min}Z=0P_{11}+5P_{12}+4P_{21}+2P_{22}+3W_{11}+4W_{12}+5W_{13}+2W_{21}+1W_{22}+2W_{23}\)约束：需求满足、产能限制、分销中心无库存 | **740000 元（最优）** |

#### （3）线性规划关键约束（必背）

* 需求约束：\(W_{11}+W_{21}=50000\)（C1 需求），\(W_{12}+W_{22}=100000\)（C2 需求），\(W_{13}+W_{23}=50000\)（C3 需求）；
* 产能约束：\(P_{21}+P_{22} \leq 60000\)（P2 产能）；
* 分销中心无库存：\(P_{11}+P_{21}=W_{11}+W_{12}+W_{13}\)，\(P_{12}+P_{22}=W_{21}+W_{22}+W_{23}\)；
* 非负约束：所有变量\(\geq 0\)。

## 六、供应链业务流程重组（流程层考点，易考阐述 / 案例）

### 1. 供应链流程 2 大分析方法

#### （1）环节法：4 大核心环节（按供应链阶段划分）

| 环节         | 涉及节点      | 核心活动                                                     |
| ------------ | ------------- | ------------------------------------------------------------ |
| 顾客订购环节 | 顾客→零售商   | 顾客抵达→订单递交→订单完成→订单接收（如用户在京东下单→京东确认→仓库发货→用户收货） |
| 补充库存环节 | 零售商→分销商 | 零售订货发起→订单递交→订单完成→订货接收（如京东门店向区域仓补货） |
| 生产环节     | 分销商→制造商 | 订单到达→生产安排→生产运输→订货接收（如区域仓向工厂下单→工厂排产→生产→送货） |
| 获取环节     | 制造商→供应商 | 原料订单发起→供应商生产→原料运输→制造商接收（如工厂向钢材供应商下单→供应商生产→送货） |

#### （2）推 / 拉法：按需求触发方式划分（结合戴尔案例，必考）

| 流程类型    | 触发条件                | 需求特点     | 戴尔案例应用                                                 |
| ----------- | ----------------------- | ------------ | ------------------------------------------------------------ |
| 推动流程    | 需求预测                | 未知、需预测 | **获取环节**：供应商生产 CPU / 内存（基于戴尔对市场需求的预测），属于推动流程 |
| 拉动流程    | 实际顾客订单            | 已知、确定   | **顾客订购 + 生产环节**：顾客下单后，戴尔组装 PC（按订单配置）、配送，属于拉动流程 |
| 推 / 拉边界 | 区分推动 / 拉动的分界点 | -            | 戴尔的**PC 机组装线起点**（组装前是推动，组装后是拉动）      |

### 2. SCOR 模型（供应链流程标准，必考阐述题）

#### （1）4 层架构（自顶向下，层层细化）

| 层次    | 名称       | 核心内容                                                     | 作用                                                         |
| ------- | ---------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 第 1 层 | 流程定义层 | 5 大核心流程：计划（P）、采购（S）、制造（M）、交付（D）、退货（R） | 确定供应链范围（如某家电企业 SCOR 流程含 P1/S1/M1/D1/R1）    |
| 第 2 层 | 流程配置层 | 26 个子流程（如 S1 = 按库存采购，M2 = 按订单制造）+3 大管理阶段（计划 / 执行 / 支持） | 构建实际供应链（如选择 “S1+M1+D1” 构成按库存销售的供应链）   |
| 第 3 层 | 流程元素层 | 拆解子流程为元素：输入 / 输出、绩效指标、最佳实践（如 S1 的输入是 “供应商清单”） | 微调运营战略（如优化 S1 的 “收货检验” 环节，降低货损率）     |
| 第 4 层 | 实施层     | 企业自定义落地细节（如 S1 的 “检验标准”“审批节点”）          | 适配企业实际情况（如某企业 S1 的检验分 “外观检验 + 性能检验” 两步） |

#### （2）5 大核心流程作用（必背）

* **Plan（计划）**：平衡总供需（如每月制定采购 / 生产 / 配送计划）；
* **Source（采购）**：获取原料 / 部件（如选择供应商、收货检验）；
* **Make（制造）**：转化原料为成品（按库存 M1 / 按订单 M2 / 按定制 M3）；
* **Deliver（交付）**：送成品到客户（如订单处理、运输、安装）；
* **Return（退货）**：处理退货（如客户退货质检、供应商退货返修）。

## 七、供应链库存控制（计算 / 案例重点，占分高）

### 1. 传统库存问题（易考简答）

| 问题           | 核心表现                                                     | 例子                                                         |
| -------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 无供应链整体观 | 各节点只关注自身库存（如零售商多订货，导致制造商库存积压）   | 某服装零售商高估需求订 1000 件，实际卖 500 件，制造商剩余 500 件库存 |
| 信息传递低效   | 信息滞后 / 失真（如分销商向制造商传订单时，人为夸大 10%）    | 牛鞭效应：顾客需求 100 件→零售商订 120 件→分销商订 150 件→制造商产 180 件 |
| 缺乏合作协调   | 节点企业间隐瞒信息（如供应商不告知产能限制，导致制造商断料） | 某芯片供应商产能不足但不告知手机厂，手机厂因缺芯片停产       |

### 2. 需求变异放大（牛鞭效应）：4 大原因（必考简答，需举例）

| 原因         | 核心逻辑                                             | 例子                                                         |
| ------------ | ---------------------------------------------------- | ------------------------------------------------------------ |
| 需求预测修正 | 下游用上游订单数据做预测，用指数平滑法修正，放大需求 | 零售商预测卖 100 台电脑，加安全库存订 120 台；批发商收到 120 台订单，修正为 150 台 |
| 订货批量决策 | 周期性订货（如每月 1 次）或凑满载，导致订单量波动    | 零售商不会卖 1 台订 1 台，而是每月汇总订 500 台，制造商收到的订单集中且量大 |
| 价格波动     | 促销（折扣 / 满减）导致 “预先购买”，订货量＞实际需求 | 超市牛奶打 8 折，零售商提前订 3 个月的货，需求降温后不再订货 |
| 短缺博弈     | 需求＞供应时按比例配给，用户故意夸大需求             | 某手机缺货时，经销商需 100 台却订 200 台，希望分到 100 台；需求缓解后订单归零 |

### 3. 库存控制模型（计算重点，分确定性 / 随机性）

#### （1）确定性模型：EOQ（经济订货批量，必考计算）

* **假设**：需求稳定、无缺货、瞬时到货、无价格折扣（基础版）；

* **核心公式**：\(EOQ = \sqrt{\frac{2AD}{rv}}\)$(\text{总相关成本（TRC）} = \sqrt{2ADrv})$（A = 固定订货成本，D = 年需求量，r = 库存持有费率，v = 单位采购成本）

* **折扣情境（全单位折扣，常考）**：步骤 1：计算无折扣 EOQ，判断是否在折扣区间；步骤 2：计算折扣点（如 Q≥100 件）的总成本（采购成本 + 订货成本 + 持有成本）；步骤 3：对比无折扣 EOQ 的总成本与折扣点总成本，选最小者。

  **例子**：D=2000 件，A=500 元，r=25%，v1=50 元（Q<100），v2=45 元（Q≥100）

  * 无折扣 EOQ=\(\sqrt{\frac{2 \times 500 \times 2000}{0.25 \times 50}}=400\)件（≥100，用 v2）；
  * 总成本 =\(2000 \times 45 + \frac{500 \times 2000}{400} + \frac{0.25 \times 45 \times 400}{2}=90000+2500+2250=94750\)元。

#### （2）随机性模型：报童模型（单周期，必考计算）

* **变量定义**：a = 零售价，b = 购进价，c = 退回价（a>b>c）；r = 随机需求量（概率 f (r)）；n = 购进量；

* **核心公式（最优购进量 n*）**：\(P(r \leq n^*) = \frac{a-b}{a-c}\)（需求量≤n*的概率 = 单位边际利润 /（单位边际利润 + 单位边际损失））

  **例子**：a=10 元，b=5 元，c=2 元，需求量 r=17-26 件（概率见文档）

  * 临界概率 =\(\frac{10-5}{10-2}=0.625\)；
  * 累加概率：P (r≤19)=0.53<0.625，P (r≤20)=0.66>0.625→n*=20 件。

### 4. 先进库存策略：VMI 与 JMI（易考案例分析）

#### （1）供应商管理库存（VMI）

| 核心要素 | 内容                                                         |
| -------- | ------------------------------------------------------------ |
| 概念     | 供应商在用户处设库存，拥有控制权（如沃尔玛的宝洁洗发水库存由宝洁管理） |
| 基本思想 | 4 大原则：合作性（共享数据）、互惠性（双方降成本）、目标一致性（统一库存目标）、持续改进 |
| 实施方法 | ①建共享信息系统（EDI/Internet）；②签框架协议（库存水平、补货策略）；③组织变革（设 VMI 协调岗） |
| 适用场景 | ①零售商无 IT 能力；②制造商实力强（如宝洁）；③直接交货水平高  |
| 支持技术 | ID 代码、条码、连续补给程序（CRP）                           |

#### （2）联合库存管理（JMI）

| 核心要素 | 内容                                                         |
| -------- | ------------------------------------------------------------ |
| 概念     | 供应商与用户联合管理库存，通过 “协调中心” 共享信息（如地区分销中心） |
| 结构模型 | 供应商→原材料联合库存→制造商→产销联合库存→分销商→顾客（协调中心统筹） |
| 优点     | ①减少牛鞭效应；②暴露供应链缺陷（如交货延迟）；③支持零库存 / 准时采购 |
| 实施策略 | ①建协调机制（共同目标、利益分配）；②用 MRPII/DRP 系统；③建快速响应系统（QR） |

| ------------------------- | ------------------------------------------------------------ | ------------------------------- | ------------------------ | ---------------------------------- |
| 直接运输                  | 供应商→需求地，无中介                                        | 无仓库成本，运输时间短          | 小需求时成本高（非满载） | 大需求（如整车送大超市）           |

| 配送中心 + 送奶线路       | 供应商→配送中心→送奶线路→需求地                              | 小批量送货成本低                | 协调更复杂               | 多小需求点（如社区便利店）         |
| 量身定做网络              | 结合多种网络（如高价值用航空 + 送奶线路，低价值用水路 + 配送中心） | 匹配产品 / 客户需求，总成本最低 | 管理复杂                 | 多产品 / 多客户（如亚马逊）        |
