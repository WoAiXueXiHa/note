# 长度==最小==的子数组

一开始的想法肯定是暴力求解：**固定左指针，右指针不断遍历，寻找符合条件的子数组**

但是**右指针每遍历一遍数组，就要回到和左指针相同位置，再次新一轮遍历，$O(n^2)$**

这种思想没有利用到题目的**正数**条件：

* 正数累加只会递增，那么**右指针就没有必要回退啊**

详细模拟一下过程：

![image-20260304121912061](https://gitee.com/binary-whispers/pic/raw/master///20260304121914412.png)

分析一下时间复杂度：最坏情况是**l和r都走到头，$O(n)$ **

思路：

* l和r两个指针维护窗口
  * l负责缩小窗口，省去r回退的步骤
  * r负责扩大窗口，不断寻找符合条件的子区间
* 条件判断
  * 外层循环：r遍历数组，不能越界->最坏的情况就是遍历完
  * 内层循环：`sum >= target`
    * r停下，l缩小区间
    * 同时更新结果

代码：

```cpp
class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
        int left = 0, right = 0;
        int n = nums.size();
        int sum = 0, len = INT_MAX; // 因为要找最小的长度，所以把len定为最大

        while(right < n) {
            // 扩大窗口
            sum += nums[right++];
            // 缩小窗口
            while(sum >= target) {
                sum -= nums[left++];
                // 更新结果
                len = min(len, right - left + 1);
            }
        }
        return (len == INT_MAX) ? 0 : len;
    }
};
```



# ==无重复字符的最长子串==

模拟过程：

![image-20260304131142129](https://gitee.com/binary-whispers/pic/raw/master///20260304131145237.png)

这里需要注意，对应字符根据ASCII码就能转成数字，就能映射到数组里的索引了

思路：

* 进窗口，r扩大窗口
  * 把遇到的新元素放入窗口，更新窗口状态
* 判断+出窗口，l缩小窗口
  * 判断：新进来的会破坏唯一性
  * 缩小：从l开始砍
    * 问题：如果是`abcb`第二个b重复了，为啥要砍a，a不是无辜的吗？
    * 回答：只要第一个b在窗口里，这个窗口就是不合法的，和新进来的b冲突了，唯一的办法就是把l的边界向右收缩，越过这个b，要越过第一个b，那么在b左边的所有字符都不能幸免，不可能属于新的窗口
* 更新，找到目前最大值
* 循环

代码：

```cpp
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        int ans = 0; // 找最大，那么定义为0
        int left = 0, right = 0;
        int n = s.size();
        vector<int> hash(128,0);

        while(right < n) {
            // 进窗口，更新哈希表映射
            hash[s[right]]++;
            // 判断，新进来的是否有冲突
            while(hash[s[right]] > 1) 
                // 出窗口，这里要把冲突的干掉，必须把第一个冲突的左边所有字符都干掉
                hash[s[left++]]--;

            // 更新结果
            ans = max(ans, right - left + 1);
            right++;    // 别忘了扩大窗口
        }
        return ans;
    }
};
```



# ==找到字符串中所有字母异位词==

思路：

![image-20260304142602343](https://gitee.com/binary-whispers/pic/raw/master///20260304142605426.png)

代码：

```cpp
class Solution {
public:
    vector<int> findAnagrams(string s, string p) {
        vector<int> pHash(26,0);    // 记录p串的哈希表
        vector<int> sHash(26,0);    // 记录s串的哈希表
        vector<int> ans;            // 返回的答案
        int m = p.size(), n = s.size();

        // 提前判断边界：s比p短，直接返回空
        if (n < m) return ans;
        // 初始化p串异位词信息并且填好第一个固定窗口
        for(int i = 0; i < m; i++) { 
            pHash[p[i] - 'a']++;
            sHash[s[i] - 'a']++;
        }
        // 看第一个窗口是否满足条件
        if(sHash == pHash) ans.push_back(0);

        for(int right = m; right < n; right++) {
            // 入窗口，吃字符
            sHash[s[right] - 'a']++;
            //出窗口，吐字符
            sHash[s[right - m] - 'a']--;
            // 比较
            if(sHash == pHash) ans.push_back(right - m + 1);
        }

        return ans;
    }
};
```

有没有优化的地方？对于两个哈希表比较，要比较26词，能不能只比较一次？

用一个`cnt`变量维护有效字符个数->真正对匹配目标字符串 `p` 产生贡献的有效字符个数

```cpp
class Solution {
public:
    vector<int> findAnagrams(string s, string p) {
        vector<int> sHash(26,0);
        vector<int> pHash(26,0);
        for(auto e : p) pHash[e - 'a']++;
        vector<int> ans;

        int n = s.size();
        int m = p.size();
        for(int left = 0, right = 0, cnt = 0; right < n; right++) {
            // 进窗口+维护cnt
            char in = s[right];
            if(++sHash[in - 'a'] <= pHash[in - 'a'])
                cnt++;
            if(right - left + 1 > m) {
                // 出窗口+维护cnt
                char out = s[left++];
                if(sHash[out - 'a']-- <= pHash[out - 'a']) 
                cnt--;
            }

            if(cnt == m) ans.push_back(left);
        }
        return ans;
    }
};
```
