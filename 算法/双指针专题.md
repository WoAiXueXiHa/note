# 移动零

本质是推动全零区间往后翻滚

![image-20260303191806336](https://gitee.com/binary-whispers/pic/raw/master///20260303191809447.png)

思路：

* 元素为零：
  * `cur++`
* 元素非零
  * `dest++`
  * 交换二者元素，此时`cur`指向元素为0，`cur++`
* 循环

代码：

```cpp
class Solution {
public:
    void moveZeroes(vector<int>& nums) {
        int dest = -1, cur = 0;
        while(cur < nums.size()){
            if(nums[cur] == 0) cur++;
            else {
                dest++;
                swap(nums[cur], nums[dest]);
                cur++;
            }
        }
    }
};
```

# 快乐数

模拟一下，发现是带环问题：

![image-20260303193139117](https://gitee.com/binary-whispers/pic/raw/master///20260303193145607.png)

思路：

1. 定义辅助函数，进行运算
2. 快指针：走两步；慢指针，走一步
3. 判断相遇时是否为1，循环条件，两指针没相遇

代码：

```cpp
class Solution {
public:
    int calculate(int n){
        int ret = 0;
        while(n){
            ret += (n % 10) * (n % 10);
            n /= 10;
        }
        return ret;
    }
    bool isHappy(int n) {
        int fast = calculate(calculate(n));
        int slow = calculate(n);

        while(fast != slow){
            fast = calculate(calculate(fast));
            slow = calculate(slow);
        }

        return fast == 1;
    }
};
```

# 复写零

模拟：

![image-20260303203749316](https://gitee.com/binary-whispers/pic/raw/master///20260303203752343.png)

**左->右会覆盖原来的值！**

思路：

`cur`指向要复写的元素，`dest`指向要复写的位置

* 找到最后一个复写的位置

  * `cur = 0` `dest = -1`
  * `cur`始终走一步
  * `dest`走的步数根据`cur`指向的元素
    * 为0：`dest`走两步
    * 非0：`dest`走一步
  * 注意`dest`不能越界

* 处理边界情况：如果最后一个要复写的是0且`dest`已经越界

  ![image-20260303204127679](https://gitee.com/binary-whispers/pic/raw/master///20260303204129531.png)

  * `arr[n - 1] = 0` `dest -= 2` `cur--`

* 从右向左开始复写

代码：

```cpp
class Solution {
public:
    void duplicateZeros(vector<int>& arr) {
        int n = arr.size();
        int dest = -1, cur = 0;

        // 1. 找到最后一个复写的位置
        while(cur < n) {
            if(arr[cur]) dest++;
            else dest += 2;
            if(dest >= n - 1) break;    // 不能越界
            cur++;
        }

        // 2. 处理边界情况，最后一个要复写的元素是0 && dest越界了 -> dest == n
        if(dest == n) {
            arr[n - 1] = 0;
            dest = n - 2;
            cur--;
        }

        // 3. 从右向左复写
        while(cur >= 0) {
            if(arr[cur]) arr[dest--] = arr[cur];
            else {
                arr[dest--] = 0;
                arr[dest--] = 0;
            }
            cur--;
        }
    }
};
```

# 三数之和

思路：对于数组内元素加减可以考虑双指针

* 排序原始数组
* 枚举一个数`nums[i]`，
  * 这里需要跳过重复的数
* 双指针区间`[i + 1， n - 1]` -> 转换成两数之和的思路
  * `sum > 0` 大了，右指针--
  * `sum < 0` 小了，左指针++
  * `sum == 0` 符合条件，插入容器
    * 这里还需要去除重复的答案

代码：

```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        sort(nums.begin(), nums.end());
        int n = nums.size();
        vector<vector<int>> ans;

        // 枚举一个数nums[i]，双指针区间[i+1,n-1]
        // 变成两数之和
        for(int i = 0; i < n - 2; i++) {
            // 过滤重复的数
            if(i > 0 && nums[i] == nums[i - 1]) continue;

            int left = i + 1, right = n - 1;
            while(left < right) {
                int sum = nums[i] + nums[left] + nums[right];
                if(sum > 0) right--;
                else if(sum < 0) left++;
                else {
                    ans.push_back({nums[i], nums[left], nums[right]});
                    // 过滤重复答案
                    while(left < right && nums[right] == nums[right - 1]) right--;
                    while(left < right && nums[left] == nums[left + 1]) left++;

                    // 寻找下一组符合条件的
                    left++;
                    right--;
                }
            } 
        }
        return ans;
    }
};
```

# 四数之和

思路和三数之和一模一样，只不过就是枚举两个数

需要注意的是加和的类型设置：

![image-20260303211518988](https://gitee.com/binary-whispers/pic/raw/master///20260303211935781.png)

![img](https://gitee.com/binary-whispers/pic/raw/master///20260303211950848)

代码：

```cpp
class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        // 先不过是枚举两个数，其它思路不变
        sort(nums.begin(), nums.end());
        int n = nums.size();
        vector<vector<int>> ans;

        for(int i = 0; i < n - 3; i++) {
            if(i > 0 && nums[i] == nums[i - 1]) continue;
            for(int j = i + 1; j < n - 2; j++) {
                if(j > i + 1 && nums[j] == nums[j - 1]) continue;

                int left = j + 1, right = n - 1;
                while(left < right) {
                    long long sum = (long long)nums[i] + nums[j] + nums[left] + nums[right];
                    
                    if(sum > target) right--;
                    else if(sum < target) left++;
                    else {
                        ans.push_back({nums[i], nums[j], nums[left], nums[right]});
                        // 除去重复答案
                        while(left < right && nums[left] == nums[left + 1]) left++;
                        while(left < right && nums[right] == nums[right - 1]) right--;

                        // 寻找下一组符合条件的组合
                        left++;
                        right--;
                    }
                }
            }
        }
        return ans;
    }
};
```

# 盛水多的容器

![image-20260303213109003](https://gitee.com/binary-whispers/pic/raw/master///20260303213112059.png)

**短板决定了盛水的上限，我现在知道了我的短板，就需要找更长的短板来弥补**

```cpp
class Solution {
public:
    int maxArea(vector<int>& height) {
        // 盛水多少的上限由短板决定
        // 数组元素加和，还是能用双指针
        int left = 0, right = height.size() - 1;
        int cur = 0, ans = 0;
        while(left < right) {
            if(height[left] < height[right]) {
                cur = height[left] * (right - left);
                ans = max(cur, ans);
                // 现在的上限是由短板决定的，所以我现在要找更长的短板
                left++;
            } else {
                cur = height[right] * (right - left);
                ans = max(ans, cur);
                right--;
            }
        }
        return ans;
    }
};
```



# 接雨水

**任意`i`位置能承载水的多少，由两边短的那面墙决定：**

即使两面墙不挨着也没关系，它周围还有水墙挡着

![image-20260303215800217](https://gitee.com/binary-whispers/pic/raw/master///20260303215802092.png)

**怎么确定两边的短板呢？**

* 如果无脑遍历，左到右和右到左同时进行，$O(n^2)$
* 可以用双指针确定：这样的时间复杂度是$O(n)$
  * left向右遍历，right向左遍历
  * leftMax和rightMax记录当前最大值

![image-20260303220432483](https://gitee.com/binary-whispers/pic/raw/master///20260303220434851.png)

**两个最大值指针怎么移动呢？**

移动较小的最大值指针，理由：

**小的指针短板上限已经确定，后续遇到再高的墙都是由短板决定；**

**而如果动大的指针，后续可能会遇到更高的墙，那么自己的短板就被抬高到大的指针了，所以大的指针那边的短板还不能确定**

![image-20260303221212717](https://gitee.com/binary-whispers/pic/raw/master///20260303221214682.png)

再总结一下思路：

1. **左指针** `left` 在头，**右指针** `right` 在尾。
2. 记录 `leftMax`（左边目前见过的最高）和 `rightMax`（右边目前见过的最高）
3. **PK 环节：**
   * 如果 `leftMax < rightMax`：说明左边是短板（木桶效应）。算出左边当前位置能装多少水，然后**左指针右移**
   * 如果 `leftMax >= rightMax`：说明右边是短板。算出右边当前位置能装多少水，然后**右指针左移**
4. 两个指针撞在一起时，结束。

代码：

```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int n = height.size();
        // 定义两个指针遍历墙
        int left = 0, right = n - 1;
        int ans = 0;
        // 找到现阶段最高的墙
        int leftMax, rightMax = 0;

        while(left < right){
            // 更新最高的墙
            leftMax = max(leftMax, height[left]);
            rightMax = max(rightMax, height[right]);

            // 一定先动短的，短的那边短板已经确定，不会改变
            // 高的那边后续短板可能抬高成现在这个指针
            if(leftMax < rightMax){
                ans += (leftMax - height[left]);
                left++;
            } else {
                ans += (rightMax - height[right]);
                right--;
            }
        }
        return ans;
    }
};
```
