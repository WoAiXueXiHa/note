## 基础思想

一张图解释：

把整个数组想象成一个 **白色丝带** ，目标是找到一条明确的分界线

* **左指针`L`及其左边** ：染成红色，**严格不满足条件** 
* **右指针`M`及其右边** ： 染成蓝色，**满足条件** 
* **`(L,R)`之间** ：白色，**未知区域** 

通过不断取中点 **M = L + (R - L)** **/** **2(防溢出)** ，判断`M`应该染成什么颜色，不断缩小白色区域。

当`L + 1 == R`时，白色区域消失，红蓝交界就是要找的答案

## 代码

```cpp
int lowerBound(vector<int>& nums, int target) {
    int left = -1, right = nums.size();
    while(left + 1 < right) {
        int mid = left + (right - left) / 2;
        // (L  TARGET  M    R)
        if(nums[mid] >= target) 
            right = mid;
        else 
            left = mid;
    }
    return right;    // left + 1
}    
```

## 举一反三

如果遇到`> < <=`又怎么处理呢？

抓住**L和R最终会紧挨着`L + 1 == R`** 这个性质

**把问题转化成”找第一个满足某条件的元素”（找蓝色的左边界）**

* `R`:**第一个** 被染蓝
* `L`：**最后一个** 被染红
* `R`和`L`紧挨着

用个表格总结：

|                             情况                             |             写法              |  如果不存在  |
| :----------------------------------------------------------: | :---------------------------: | :----------: |
|                      第一个`>= x`的位置                      |     `lowerBound(nums, x)`     | 返回数组长度 |
|          第一个`> x`的位置->第一个`>= x + 1`的位置           |   `lowerBound(nums, x + 1)`   | 返回数组长度 |
|       最后一个`< x`的位置->`>= x`的数的左边第一个位置        |   `lowerBound(nums, x + 1)`   |    返回-1    |
| 最后一个`<= x`的位置->`> x`的数的左边第一个位置->`>= x + 1`的数的左边第一个位置 | `lowerBound(nums, x + 1) - 1` |    返回-1    |
