# 相向双指针（核心：利用单调性，降维排除无效解）

* **[167. 两数之和 II]**：有序数组首尾相加，和大于目标降右（`right--`），小于目标升左（`left++`），注意返回值的大坑
* **[15. 三数之和]**：先排序，外层 `for` 固定一端，内层双指针逼近，注意`i`要跳过重复值，答案入数组后也要跳过重复的答案
* **[2824.  统计和小于目标的下标对数目]**：先排序，双指针逼近，小于`target`，整个区间都满足条件，判断要不要+1，取特殊值
* **[16. 最接近的三数之和]**：排序定基准，双指针逼近目标值，实时 `ans = 最小差值`。
* **[18. 四数之和]**：先排序，固定枚举两个数，剩下两个数作为双指针逼近，也别忘了i和j的重复判断和答案重复判断
* **[2824. 统计和小于目标的下标对]**：排序后双指针，若 `sum < target`，则当前 `left` 与 `right` 之间的**所有组合均合法**，直接累加 `right - left` 并 `left++`
* **[611. 有效三角形的个数]**：排序后固定最长边在**最右侧**，左侧两指针相向逼近，两小边之和 $> $ 第三边则中间区间全部成立
* **[11. 盛最多水的容器]**：木桶效应，容积受限于短板，收缩较矮的一侧寻找最优答案
* **[42. 接雨水 (Hard)]**：双指针动态维护左右两侧历史最高，哪侧短板低就直接结算哪侧坑位水量
* **[125. 验证回文串]**：首尾向内收缩，用`isalnum`跳过非字母数字字符，统一大小写后比对
* **[2105. 给植物浇水 II]**：纯物理模拟，首尾指针各自带水壶向中间推进，水量不足就加满，相遇时就看ans到底要不要+1（当前植物是否大于二者最大）

#  滑动窗口（同向双指针）（核心：右指针开拓找合法，左指针收缩榨水分）

**最小循环里更新，最大循环外更新**

* **[209. 长度最小的子数组]**：右扩累加，超标后触发 `while`，**进门先算最小长度**，然后左侧持续减去元素
* **[3. 无重复字符的最长子串]**：题目中的字符涵盖了ASCII，数组模拟哈希表，右扩进哈希表，查到频次 $>1$ 则触发左侧无脑出窗口，**出门再算最大长度**
* **[713. 乘积小于 K 的子数组]**：特判 `k<=1`返回0，右扩相乘，超标左缩除掉，合法区间内新增的子数组数量恰为 `right - left + 1`
* **[3090. 最多两次的最长子串] / [2958. 最多 K 个的最长子数组]**：两题逻辑完全一致，右侧吃进更新频次，只要当前字符频次 $> 2$（或 $> K$），左指针就一直吐出，直到频次达标，最后出门算最长
* **[2730. 最长的半重复子串]**：维护窗口内“相邻且相同”的字符对数量，这对数一旦 $>1$，左侧收缩直到吐出一对
* **[2779. 数组的最大美丽值]**：排序后降维，转化为求一个最长的滑动窗口，保证窗口内 `最大值 - 最小值 <= 2*k`
* **[1004. 最大连续 1 的个数 III]**：把翻转 $0$ 视为消耗配额，遇到 $0$ 扣额度，额度透支则左指针右移找 $0$ 吐出以恢复额度。
* **[2962. 最大元素至少 K 次的子数组]**：先扫一遍找全局最大值，滑动窗口记录该值频次，达标后，左指针左侧的所有元素拼上当前窗口**均合法**。
* **[2302. 得分小于 K 的子数组]**：右扩更新和，计算当前得分 `sum * length`，超标则左缩，累加合法长度。
* **[1658. 将 x 减到 0 的最小操作数]**：逆向思维，把“两端切除”转化为“求中间一段连续的最长滑动窗口”，使其和严格等于 `总和 - x`。
* **[1234. 替换子串得到平衡]**：逆向判定，关注“窗口外”，只要窗口外的所有字符频次均 $\le n/4$，该窗口即为合法万能牌，进门算最短。
* **[76. 最小覆盖子串 (Hard)]**：两张哈希表打配合，右侧贪心吃进直到满足所有需求；触发 `while` 后进门算最短，左侧疯狂吐出冗余字符榨干水分。
